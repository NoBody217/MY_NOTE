# 1.Vim是什么
- Vim是编辑器，不是编译器
	- 编辑器只负责对文本进行编辑，而不负责编译，调试
	- 其实，VS code也是编辑器，我们能使用他进行编译，运行只是因为集成的插件帮我们完成了这件事情
- Vim哲学
	- 对于一个程序员，写代码只是短暂的过程，我们更多的时间会花在阅读代码，编辑一些细节
	- 对于Vim使用者来说，鼠标是多余的
	- Vim通过不同模式的切换完成插入文本和操作文本
	- Vim本身是一种编程语言，按键是命令，命令是可组合的
	- Vim的最终目的是一个能匹配你思考速度的编辑器
# 2.模态编辑
- Vim主要有如下几种操作模式
	- Normal：普通模式，用于在文件中移动和进行编辑
	- Insert：插入模式，用于插入文本，类似传统的编辑器
	- Replace：替换模式，用于替换文本，直接覆写现有字符
	- Visual：可视化模式，用于选择文本块
	- Command-lin：命令行模式，用于运行各种命令
- **模式切换**：从普通模式
	- 按 `i` 进入**插入模式** (在光标前插入)
	- 按 `I` 进入**插入模式** (行首非空白处)
	- 按 `a` 进入**插入模式** (在光标后)
	- 按 `A` 进入**插入模式** (行尾)
	- 按 `o` 进入**插入模式** (在下方新行)
	- 按 `O` 进入**插入模式** (在上方新行)
	- 按 `R` 进入**替换模式**
	- 按 `v` 进入**可视化模式** (字符)
	- 按 `V` 进入**可视化行模式**
	- 按 `<C-v>` 进入**可视化块模式**
	- 按 `:` 进入**命令行模式**
# 3.基础
- 缓冲区(buffers)标签页(tabs)和窗口(windows)
	- Vim维护一组打开的文件，称为缓冲区
	- 一个Vim会话可以有多个标签页
	- 每个标签页可以有多个窗口(分割的窗格)
	- 每个窗口仅显示一个缓冲区
	- 窗口仅仅是视图，一个缓冲区可以在多个窗口中打开，来查看一个文件的不同部分。
- 缓冲区、窗口、标签页管理技巧:
	- 缓冲区:
		- `:ls` / `:buffers` 列出
		- `:b <N>` / `:b <name>` 切换
		- `:bn` / `:bp` 下一个/上一个
		- `:bd` 关闭
	- 窗口 (分割):
		- `:sp <file>` / `Ctrl-w s` 水平分割
		- `:vsp <file>` / `Ctrl-w v` 垂直分割
		- `Ctrl-w w` / `Ctrl-w hjkl` 切换
		- `Ctrl-w q` / `:q` 关闭
		- `Ctrl-w =` / `_` / `|` 调整大小
	- 标签页:
		- `:tabnew <file>` / `:tabe <file>` 新建
		- `gt` / `gT` 下一个/上一个
		- `Ngt` 跳转到第N个
		- `:tabc` 关闭
		- `:tabs` 列出
- 命令行模式 (Command-line)
	- 在普通模式下键入 : 可以进入命令行模式。光标会跳转到屏幕底部的命令行。此模式用于执行许多命令：
		-  `:q`: 退出 (关闭窗口)
		- `:q!`: 强制退出 (丢弃更改)
		- `:w`: 保存 (写入)
		- `:wq` / `:x`: 保存并退出
		- `:wq!`: 强制保存并退出
		- `:e {文件名}`: 打开文件
		- `:e!`: 重新加载当前文件
		- `:ls`: 显示打开的缓冲区
		- `:help {主题}`: 打开帮助
		- `:helpgrep <pattern>`: 搜索帮助
# 4.移动
## 基本移动
- `h` 左
- `j` 下
- `k` 上
- `l` 右
## 单词移动
- `w / W` 下一个单词/大写单词开头
- `b / B` 上一个单词/大写单词开头
- `e / E` 当前/下一个单词/大写单词结尾
## 行内移动
- `0` 行首 (第一列)
- `^` 行首 (第一个非空白)
- `$` 行尾
- `g_` 行尾 (最后一个非空白)
- `+` / Enter 下一行首非空白
- `-` 上一行首非空白
## 屏幕移动
- `H` 屏幕顶部行
- `M` 屏幕中间行
- `L` 屏幕底部行
## 滚动
- `Ctrl-u / Ctrl-d` 上/下滚动半屏
- `Ctrl-f / Ctrl-b` 上/下滚动整屏
- `zt / zz / zb` 当前行滚动到顶/中/底
## 文件跳转
- `gg` 文件首行
- `G` 文件末行
- `{行号}G` 或 :{行号} 跳转指定行
- `Ctrl-g` 显示文件信息
- `%` 跳转到匹配括号/注释等
****## 行内查找 (Find)
- `f{char} / F{char}` 向前/后查找字符
- `t{char} / T{char}` 向前/后查找到字符前/后
- `; / ,` 重复/反向重复查找
## 文件搜索 (Search)
- `/{regex} / ?{regex}` 向下/上搜索
- `n / N` 下一个/上一个匹配
- `* / #` 向下/上搜索光标处单词
- `:set hlsearch` 高亮匹配
# 5.编辑
## 删除/更改
- `d{motion}` 删除 (例: dw, d$, dd)
- `c{motion}` 更改 (删除并插入) (例: cw, c$, cc)
- `x` 删除光标处字符
- `X` 删除光标前字符
- `s` 替换光标处字符并插入
- `S` 删除当前行并插入
## 复制/粘贴
- `y{motion}` 复制/Yank (例: yw, yy)
- `p / P` 在光标后/前粘贴
- 可视化模式下 `y` 复制选中
- 可视化模式下 `d / c` 剪切选中
## 其他编辑
- `o / O `下/上插入新行
- `u` 撤销
- `Ctrl-r` 重做
- `.` 重复上次更改
- `~` 翻转大小写
# 6.计数 (Counts)
可以将数字（计数）与动词和名词组合，重复执行操作。
- `3w` - 向前移动3个单词。
- `5j` - 向下移动5行。
- `d3w` 或 `3dw` - 删除3个单词。
- `3dd` - 删除3行。
- `c5w` - 修改5个单词。
# 7.文本对象
- 文本对象允许更精确地指定操作范围，通常由 i (inner) 或 a (around) 后跟一个边界字符组成。
## 括号/引号
- `ci( / ca(`: 更改 () 内部/周围
- `di[ / da[`: 删除 \[] 内部/周围
- `yi{ / ya{`: 复制 {} 内部/周围
- `ci" / ca"`: 更改 "" 内部/周围
- `di' / da'`: 删除 '' 内部/周围
## 词/句/段
- `ciw / caw`: 更改 内部/周围 单词
- `cis / cas`: 更改 内部/周围 句子
- `cip / cap`: 更改 内部/周围 段落
## 标签
- `cit`: 更改标签内部
- `dat`: 删除标签及内容
示例：dap (删除整个段落), yit (复制标签内文本), v2as (选择两个句子)。
# 8.自定义
- Vim 通过一个位于 `~/.vimrc` 的纯文本配置文件（包含 Vimscript 命令）进行高度自定义
# 9.插件
- 有大量的插件可以扩展 Vim。从 Vim 8.0 开始，推荐使用内置的**包管理系统 (package management system)**，而不是第三方插件管理器。
- 只需创建目录 `~/.vim/pack/vendor/start/` (或类似名称)，然后将插件放在那里（例如通过 `git clone`）。
